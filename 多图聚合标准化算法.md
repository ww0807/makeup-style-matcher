# 风格DNA · 多图聚合标准化算法

> 版本: v1.0 | 日期: 2026-05-31
> 解决问题: 同风格多图分析后，各维度枚举值离散时如何收敛为标准化公式

---

## 一、问题定义

### 现状

用户上传一组「同风格」的多张图片（如 6 张韩系Clean Fit 照片），经 VLM 单图分析后再做**置信度加权投票聚合**，结果如下：

```
上衣类型维度:
  图1: T恤        (conf 0.82) → 权重 27%
  图2: 吊带/背心   (conf 0.78) → 权重 25%
  图3: 衬衫       (conf 0.80) → 权重 26%
  图4: 西装外套    (conf 0.75) → 权重 22%

输出: T恤 27% · 背心 25% · 衬衫 26% · 西装外套 22%
```

**问题**: 四分五裂的碎片化分布，无法给出有意义的"标准答案"。但用户认为这是同一套风格，应该能收敛出一个标准化的风格公式。

### 根本原因

18 个维度的**性质不同**——有些是"风格指纹"（同风格几乎不变），有些是"单品选择"（同风格内天然多样）。用统一的投票法忽略了这种差异。

## 三、分级判定算法

### 3.1 集中度计算

对每个维度的 `distribution`（已有置信度加权排序后的结果），计算两个指标：

```javascript
/**
 * 计算维度的集中度指标
 * @param {Array} distribution - 已按 weight 降序排列的分布数组
 *                                [{value, weight}, ...]
 * @returns {Object} { top1, entropy }
 *   - top1: 第一名占比 [0,1]
 *   - entropy: 归一化信息熵 [0,1]，0=最集中，1=最均匀分散
 */
function calcConcentration(distribution) {
  if (!distribution.length) return { top1: 0, entropy: 0 };

  const top1 = distribution[0].weight;

  // 信息熵 H = -Σ(p_i × log₂(p_i))
  let entropy = 0;
  for (const d of distribution) {
    if (d.weight > 0) {
      entropy -= d.weight * Math.log2(d.weight);
    }
  }

  // 归一化到 [0,1]: 除以最大可能熵值 log₂(n)
  const maxEntropy = Math.log2(distribution.length);
  const normEntropy = maxEntropy > 0 ? entropy / maxEntropy : 0;

  return { top1, entropy: normEntropy };
}
```

**信息熵直觉理解**：

```
分布A: [0.72, 0.15, 0.08, 0.05] → 熵 ≈ 0.28 → 很集中 → A类
分布B: [0.42, 0.33, 0.15, 0.10] → 熵 ≈ 0.58 → 中等   → B类
分布C: [0.27, 0.26, 0.25, 0.22] → 熵 ≈ 0.88 → 很均匀   → C类
```

### 3.2 分级规则

```javascript
/**
 * 根据集中度指标和样本量判定维度级别
 * @param {Object} concentration - { top1, entropy }
 * @param {number} visibleCount - 有效识别该维度的图片数
 * @returns {Object} { level, label, reason }
 */
function classifyDimension(concentration, visibleCount) {
  const { top1, entropy } = concentration;

  // 样本不足时默认归入 A 类（保守策略）
  if (visibleCount < 3) {
    return {
      level: 'A',
      label: 'core',
      reason: `样本不足(${visibleCount}张)，默认核心特征`,
      confidence: 'low'
    };
  }

  // ── A类: 高一致 ──
  // Top1 占比 ≥ 55%，且分布不够均匀（熵 < 0.5）
  // 含义: 绝大多数图片选择了同一个值
  if (top1 >= 0.55 && entropy < 0.5) {
    return {
      level: 'A',
      label: 'core',
      reason: `高度集中 top1=${(top1 * 100).toFixed(0)}%`,
      confidence: 'high'
    };
  }

  // ── B类: 有限变体 ──
  // Top1 在 35%~55% 之间，或 熵在 0.5~0.75
  // 含义: 有一个主选项，但备选项也有显著占比
  if (top1 >= 0.35 && entropy < 0.75) {
    return {
      level: 'B',
      label: 'variant',
      reason: `有限变体 top1=${(top1 * 100).toFixed(0)}%`,
      confidence: 'medium'
    };
  }

  // ── C类: 高离散 ──
  // 其他情况: Top1 < 35% 或 熵 ≥ 0.75
  // 含义: 分布很散，没有明显的主导选项
  return {
    level: 'C',
    label: 'scattered',
    reason: `离散分布 top1=${(top1 * 100).toFixed(0)}%, 熵=${entropy.toFixed(2)}`,
    confidence: 'low'
  };
}
```

### 3.3 判定阈值可视化

```
                    熵(Entropy)
                     ^
                     |
              0.75   |...........B类区域................
                     |         .    .
                     |      .         .
              0.50   |....B.........A类...............
                     |   .            .
                     | .                .
               0.00   +----+----+----+---> Top1占比
                     0   0.25 0.35 0.55  1.00
                     
                     左下角(A): top1大+熵小 = 集中 ✓
                     中间(B):  top1中等+熵中等 = 有限变体
                     右上(C):  top1小+熵大 = 离散 ✗
```

### 3.4 特殊覆盖规则

上述自动判定可被**预定义的分类映射**覆盖（因为某些维度的本质决定了它必然属于某类）：

```javascript
// 预定义强制分类（基于领域知识）
const DIMENSION_CLASS_OVERRIDE = {
  // 强制 A 类 — 这些维度天然是风格指纹
  'makeup_style':  'A',   // 化妆风格: 换了就是换风格
  'lip_style':     'A',   // 唇妆风格
  'base_texture':  'A',   // 底妆质感
  'brow_style':    'A',   // 眉妆风格
  'hair_color':    'A',   // 发色
  'color_tone':    'A',   // 色彩倾向
  'filter_tone':   'A',   // 滤镜色调

  // 强制 C 类 — 这些维度天然是单品实例
  'top_type':      'C',   // 上衣类型: 每张图衣服不同
  'bottom_type':   'C',   // 下装类型
  'accessory':     'C',   // 核心配饰

  // B 类或不强制 — 让算法自动判断
  // hair_shape, bangs, outfit_style, skin_expose
  // scene, light, expression, interaction, framing
};
```

**最终判定逻辑**：
```javascript
function finalClassification(dimKey, autoResult) {
  const forced = DIMENSION_CLASS_OVERRIDE[dimKey];
  if (forced) {
    return {
      ...autoResult,
      level: forced,
      label: forced === 'A' ? 'core' : forced === 'B' ? 'variant' : 'scattered',
      reason: autoResult.reason + ' [预定义]'
    };
  }
  return autoResult; // 用自动判定的结果
}
```

## 四、各级别的聚合策略

### 4.1 A 类策略：Top1 单值

**逻辑**: 最简单的——直接取分布第一名作为标准答案。

```javascript
function aggregateClassA(dimKey, dimLabel, aggData) {
  const d = aggData[dimKey];
  const top = d.distribution[0];

  return {
    dimension: dimKey,
    label: dimLabel,
    level: 'A',
    displayMode: 'single',

    // 标准答案
    consensus: top.value,

    // 统计支撑
    confidence: top.weight,          // 该值占总权重比例
    support: `${d.visibleCount}张`,   // 多少张图有效识别
    avgConfidence: d.avgConfidence,   // 平均模型置信度

    // 原始数据保留（供展开查看）
    rawDistribution: d.distribution,
    samples: d.samples
  };
}
```

**输出示例**:
```json
{
  "dimension": "makeup_style",
  "label": "化妆风格",
  "level": "A",
  "displayMode": "single",
  "consensus": "清冷妆",
  "confidence": 0.72,
  "support": "6张",
  "avgConfidence": 0.78
}
```

### 4.2 B 类策略：Top-N 集合

**逻辑**: 取前 N 个值，直到累计占比达到阈值（默认 80%），形成「主推 + 备选」的结构。

```javascript
/**
 * B类聚合：获取共识集合
 * @param {Array} distribution - 排序后的分布
 * @param {number} threshold - 累积占比阈值，默认 0.80
 * @returns {Object} { primary, alternatives, cumulative }
 */
function aggregateClassB(dimKey, dimLabel, aggData, threshold = 0.80) {
  const d = aggData[dimKey];

  // 取 top-N 直到累积超过阈值
  let cumul = 0;
  const selected = [];
  for (const item of d.distribution) {
    selected.push({ value: item.value, weight: item.weight });
    cumul += item.weight;
    if (cumul >= threshold) break;
    // 安全边界：最多取 5 个
    if (selected.length >= 5) break;
  }

  return {
    dimension: dimKey,
    label: dimLabel,
    level: 'B',
    displayMode: 'multi',

    // 主推（排名第一）
    primary: selected[0].value,
    primaryWeight: selected[0].weight,

    // 共识集合（所有选中的）
    consensus: selected.map(s => `${s.value}`),

    // 各项权重详情
    alternatives: selected.map(s => ({
      value: s.value,
      weight: s.weight,
      pct: `${(s.weight * 100).toFixed(0)}%`
    })),

    // 累积占比
    cumulative: Math.min(cumul, 1),
    support: `${d.visibleCount}张`,

    // 原始数据
    rawDistribution: d.distribution,
    samples: d.samples
  };
}
```

**输出示例**:
```json
{
  "dimension": "hair_shape",
  "label": "发型轮廓",
  "level": "B",
  "displayMode": "multi",
  "primary": "长直发",
  "primaryWeight": 0.42,
  "consensus": ["长直发", "长卷发"],
  "alternatives": [
    {"value": "长直发", "weight": 0.42, "pct": "42%"},
    {"value": "长卷发", "weight": 0.33, "pct": "33%"}
  ],
  "cumulative": 0.75,
  "support": "6张"
}
```

### 4.3 C 类策略：LLM 语义归纳

**这是整个方案的核心创新点。**

#### 4.3.1 问题分析

C 类维度（上衣、下装、配饰、露肤）的每个选项都是具体的**单品名称**：
- 上衣: T恤 / 衬衫 / 吊带 / 西装 / 针织卫衣 / 皮衣 ...
- 下装: 牛仔裤 / 阔腿裤 / A字裙 / 包臀裙 / 工装裤 ...
- 配饰: 帽子 / 墨镜 / 项链 / 耳环 / 手表 / 包袋 ...

这些选项虽然字面完全不同，但在**风格语义空间中有隐含的共同点**：
- T恤、衬衫、吊带、西装 → 可能共同点是「修身/利落剪裁」
- 牛仔裤、阔腿裤、A字裙 → 可能共同点是「休闲日常」

#### 4.3.2 解决思路

**两步走**:

1. **共性归纳 (commonTrait)**: 用一句话提炼这些不同单品的风格共同点
2. **标准标签选取 (standardLabel)**: 从合法枚举中选 1 个最具代表性的标签作为该风格的"标准答案"

#### 4.3.3 Prompt 设计

```javascript
/**
 * 构建C类维度的LLM归纳Prompt
 */
function buildScatteredDimPrompt(dimKey, dimLabel, distribution, allValues, imageCount) {

  const distLines = distribution
    .map((d, i) => `  ${i + 1}. ${d.value} (占比 ${(d.weight * 100).toFixed(0)}%)`)
    .join('\n');

  const enumText = allValues.join(' | ');

  return `你是一位资深形象顾问。以下是某人上传的一组「${imageCount}张同风格照片」中，「${dimLabel}」这个维度的AI分析汇总。

【分析结果】（按出现频率排序）:
${distLines}

注意：以上结果是来自${imageCount}张属于同一个风格的图片。也就是说，这个人在这个风格下会穿/使用以上多种不同的${dimLabel}。

【${dimLabel}的所有合法选项】:
${enumText}

请完成以下任务：

1. 【提炼共同特征】
   以上不同选项之间有什么隐含的风格共同点？
   要求：用一句话概括（不超过20字），聚焦于风格属性（如剪裁/质感/色调/场合感），不要逐个列举。
   
2. 【选出标准标签】
   从上述「所有合法选项」中，选出 1 个最能代表这种共同特征的**标准标签**。
   - 如果某个单一标签就能很好地代表这个群体，直接输出它
   - 如果没有单一的完美代表（比如同时包含裙子和裤子），则输出 "MULTI"，并在 altLabels 中列出 2-3 个代表

3. 【给出理由】
   简述选择理由（不超过25字）

严格输出 JSON，不要 markdown 包裹，不要解释文字：
{
  "commonTrait": "20字以内的共同特征描述",
  "standardLabel": "单个枚举值字符串 或 MULTI 字符串",
  "altLabels": ["备选标签1", "备选标签2"],
  "reasoning": "25字以内的选择理由"
}`;
}
```

#### 4.3.4 调用与解析

```javascript
async function summarizeScatteredDim(dimKey, dimLabel, distribution, allValues, imageCount) {
  const prompt = buildScatteredDimPrompt(dimKey, dimLabel, distribution, allValues, imageCount);

  try {
    const rawResponse = await callLLMText(prompt);
    
    const cleaned = rawResponse
      .replace(/^```json\s*/i, '')
      .replace(/^```\s*/, '')
      .replace(/```\s*$/, '')
      .trim();

    let parsed;
    try {
      parsed = JSON.parse(cleaned);
    } catch (e) {
      const match = cleaned.match(/\{[\s\S]*\}/);
      if (match) parsed = JSON.parse(match[0]);
      else throw new Error('无法解析JSON');
    }

    return {
      commonTrait: parsed.commonTrait || '',
      standardLabel: parsed.standardLabel || distribution[0]?.value || '',
      altLabels: Array.isArray(parsed.altLabels) ? parsed.altLabels : [],
      reasoning: parsed.reasoning || ''
    };

  } catch (error) {
    console.warn(`[C类归纳失败] ${dimKey}: ${error.message}`);
    return {
      commonTrait: '(归纳暂不可用)',
      standardLabel: distribution[0]?.value || '',
      altLabels: distribution.slice(1, 3).map(d => d.value),
      reasoning: '降级: 使用最高频值',
      error: error.message
    };
  }
}
```

#### 4.3.5 批量优化：合并请求

C 类通常有 3~4 个维度。为了减少 API 调用次数和延迟，可以将多个 C 类维度**合并在一个 Prompt 中**：

```javascript
/**
 * 批量处理所有C类维度（一次LLM调用处理多个维度）
 */
async function summarizeAllScatteredDims(scatteredDims, imageCount) {
  
  const dimsBlock = scatteredDims.map((d, idx) => {
    const distLines = d.distribution
      .map((item, i) => `    ${i + 1}. ${item.value} (${(item.weight * 100).toFixed(0)}%)`)
      .join('\n');
    const enumLine = d.allValues.join(' | ');
    return `维度${idx + 1}: ${d.dimLabel} (${d.dimKey})
  合法选项: ${enumLine}
  分析结果:
${distLines}`;
  }).join('\n\n');

  const prompt = `你是资深形象顾问。以下是一组同风格照片中，几个"单品类"维度的AI分析汇总。

这些维度每张图的实物不同是正常的（如同一个人不会总穿同一件T恤），但它们应该有风格上的共同点。

【各维度数据】
${dimsBlock}

请对每个维度分别输出归纳结果：

严格输出 JSON 数组（顺序对应上面的维度1/2/3...），不要markdown包裹：
[
  {
    "commonTrait": "共同特征（20字以内）",
    "standardLabel": "标准标签（必须是上面给出的合法选项之一）或 MULTI",
    "altLabels": ["备选1", "备选2"],
    "reasoning": "理由（25字以内）"
  }
]`;

  try {
    const raw = await callLLMText(prompt);
    const cleaned = raw.replace(/^```\w*\s*|^```|\s*```$/g, '').trim();
    const results = JSON.parse(cleaned);

    const map = new Map();
    scatteredDims.forEach((d, idx) => {
      const r = results[idx] || {};
      map.set(d.dimKey, {
        commonTrait: r.commonTrait || '',
        standardLabel: r.standardLabel || d.distribution[0]?.value || '',
        altLabels: Array.isArray(r.altLabels) ? r.altLabels : [],
        reasoning: r.reasoning || ''
      });
    });

    return map;

  } catch (error) {
    console.warn('[批量C类归纳失败]', error.message);
    const fallback = new Map();
    scatteredDims.forEach(d => {
      fallback.set(d.dimKey, {
        commonTrait: '(归纳不可用)',
        standardLabel: d.distribution[0]?.value || '',
        altLabels: d.distribution.slice(1, 3).map(x => x.value),
        reasoning: '降级: 最高频值'
      });
    });
    return fallback;
  }
}
```

### 5.2 主函数实现

```javascript
/**
 * 标准化聚合入口
 * @param {Object} agg - 现有 aggregate() 的返回值
 * @param {number} imageCount - 有效图片总数
 * @returns {Promise<Object>} standardizedResult
 */
async function standardizeAggregate(agg, imageCount) {
  const result = {
    dimensions: {},
    byGroup: {},
    classStats: { A: 0, B: 0, C: 0 },
    summary: ''
  };

  // 收集 C 类维度（用于批量 LLM 调用）
  const cClassDims = [];

  for (const dim of STYLE_DIMENSIONS) {
    const d = agg[dim.key];
    if (!d || !d.distribution?.length) continue;

    // Step 1: 计算集中度
    const conc = calcConcentration(d.distribution);

    // Step 2: 分级判定
    let cls = classifyDimension(conc, d.visibleCount);
    cls = finalClassification(dim.key, cls);
    result.classStats[cls.level]++;

    let dimResult;

    if (cls.level === 'A') {
      dimResult = aggregateClassA(dim.key, dim.label, agg);
    } else if (cls.level === 'B') {
      dimResult = aggregateClassB(dim.key, dim.label, agg);
    } else {
      cClassDims.push({
        dimKey: dim.key,
        dimLabel: dim.label,
        distribution: d.distribution,
        allValues: dim.values || getAllSubStyleValues(dim),
        aggData: d
      });

      // 占位，等 LLM 返回后填充
      dimResult = {
        dimension: dim.key,
        label: dim.label,
        level: 'C',
        displayMode: 'summarized',
        status: 'pending',
        consensus: '-',
        commonTrait: 'AI 分析中...',
        rawDistribution: d.distribution,
        support: `${d.visibleCount}张`
      };
    }

    dimResult._classInfo = cls;
    result.dimensions[dim.key] = dimResult;
    (result.byGroup[dim.group] ||= []).push(dimResult);
  }

  // 批量处理 C 类维度
  if (cClassDims.length > 0) {
    const cResults = await summarizeAllScatteredDims(cClassDims, imageCount);

    for (const cd of cClassDims) {
      const summary = cResults.get(cd.dimKey);
      const assembled = assembleClassCResult(cd.dimKey, cd.dimLabel, cd.aggData, summary);
      assembled._classInfo = result.dimensions[cd.dimKey]._classInfo;
      result.dimensions[cd.dimKey] = assembled;

      const groupArr = result.byGroup[
        STYLE_DIMENSIONS.find(d => d.key === cd.dimKey)?.group
      ];
      if (groupArr) {
        const idx = groupArr.findIndex(x => x.dimension === cd.dimKey);
        if (idx >= 0) groupArr[idx] = assembled;
      }
    }
  }

  // 生成一句话摘要
  result.summary = buildOneLineFormula(result);

  return result;
}

// 辅助：获取分层维度的所有细分值
function getAllSubStyleValues(dim) {
  if (!dim.layered || !dim.subStyles) return dim.values || [];
  const all = [];
  for (const subs of Object.values(dim.subStyles)) {
    all.push(...subs);
  }
  return all;
}
```

### 5.3 一句话公式生成

```javascript
/**
 * 从标准化结果生成一行简洁的DNA公式
 */
function buildOneLineFormula(stdResult) {
  const parts = [];
  const groupOrder = ['妆容', '发型', '穿搭', '配饰', '氛围'];

  for (const g of groupOrder) {
    const dims = stdResult.byGroup[g] || [];
    if (!dims.length) continue;

    const items = dims.map(d => {
      if (d.level === 'A' && d.consensus) return d.consensus;
      else if (d.level === 'B' && d.primary) return d.primary;
      else if (d.level === 'C' && d.consensus && d.consensus !== '-') return d.consensus;
      return '?';
    }).filter(Boolean);

    if (items.length) parts.push(items.join('·'));
  }

  return parts.join(' × ');
}

## 七、性能与体验优化

### 渐进式渲染时间线

```
t=0ms     用户点击"开始分析"
t=~500ms  开始调 VLM 分析各图
t=~8s     所有图分析完成
t=8.01ms  现有 aggregate() → 立即显示旧版结果
t=8.02ms  新增: A/B 类维度本地计算完成 → 立即渲染
          C 类维度显示 "AI 分析中..." 占位
t=~11ms   C 类 LLM 返回 → 填充 C 类维度 → 更新 UI
t=~13ms   人设解读生成完成（并行）
t=~13ms   全部完成
```

**关键**: A/B 类零延迟，C 类异步加载不阻塞整体展示。

### 缓存策略

同一批图片不重复请求 C 类归纳（基于图片文件名 hash 判断是否变化）。

### 错误降级

| 失败场景 | 降级策略 |
|---------|---------|
| LLM 调用超时 | C 类维度显示 Top1 + 标注"(未归纳)" |
| LLM 返回非法 JSON | 正则提取兜底 → 再失败则用 Top1 |
| LLM 返回值不在枚举库内 | 后验校验，回退到分布第一名的合法值 |
| 图片数 < 3 | 不触发 C 类归纳，全部按 A/B 处理 |
